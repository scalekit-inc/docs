Error Handling

If there are any configuration or application issues while the user tries to login with Single Sign-on via Scalekit, your application's redirect uri will receive the details of the error via the request parameters: error_code, error_description.

Error CodeError DescriptionPossible Resolution Strategy
invalid_redirect_uriRedirect URI is not part of the pre-approved list of Redirect URIs. If this is a valid URL, you need to add it in the Scalekit Dashboard before using it.
invalid_organization_idOrganization ID is not valid
invalid_connection_selectorAtleast one of organization_id, connection_id, domain need to be present in the Authorization URL request to process the Single Sign-on request
no_active_connectionsThere are no active SSO Connections configured to process the Single Sign-on request
connection_not_activeConnection ID requested is not yet enabled to process login for users.This connection needs to be enabled before it can process login for real users.
no_configured_connectionsThere are no active SSO Connections configured to process the Single Sign-on request
invalid_organization_idInvalid Organization ID requested to process the Single Sign-on request
invalid_connection_idInvalid Connection ID requested to process the Single Sign-on request
domain_not_foundThere are no active SSO Connections configured for this domain to process the Single Sign-on request
invalid_user_domainUser doesn't belong to the allowed domains list for this SSO Connection.
mandatory_attribute_missingMandatory attributes are missing
